## Project Overview

**Pushp Events Menu Generator** is a desktop application that generates bilingual (English/Hindi) menu PDFs for multi-day catering events. It reads structured event and menu data from Excel files and produces formatted PDF menus.

## Commands

### Development Setup
```bash
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Run
```bash
python app.py                    # GUI mode (primary)
python generate_menu.py [path/to/file.xlsx]  # CLI mode
```

### Build Executables
```bash
# macOS
./build_mac.sh
# Output: dist/Pushp-Menu-Generator.app

# Windows
build_windows.bat
# Output: dist/Pushp-Menu-Generator.exe
```

## Architecture

### Data Flow
```
Excel File (.xlsx)
  ├── event_info sheet  → read_event_info()
  ├── menu sheet        → read_menu_data()
  └── meal_counts sheet → read_meal_counts()
          │
          ▼
  menu_generator.py
  ├── build_menu_tree()    → date → meal → category → items[]
  └── build_date_pages()  → page objects for layout
          │
          ▼
  Jinja2 (templates/menu.html) + WeasyPrint
          │
          ▼
  Generated-menu/<excel_name>/<event_name>_English.pdf
                             /<event_name>_Hindi.pdf
```

### Key Files

| File | Role |
|------|------|
| `app.py` | Tkinter GUI — file selection, generate/reset buttons, GTK3 config on Windows |
| `menu_generator.py` | All core logic: Excel I/O, data normalization, menu building, PDF generation (~840 lines) |
| `generate_menu.py` | Thin CLI wrapper around `menu_generator.py` |
| `templates/menu.html` | Jinja2 template — cover page, date pages, bilingual layout |

### menu_generator.py Internals

**Constants:** `MEAL_ORDER`, `MEAL_LABELS_EN/HI`, `CATEGORY_LABELS_HI`, `LABELS_EN/HI`, `MONTHS_HI`, `EVENT_KEY_ALIASES`

**Data normalization:** `normalize_key()`, `normalize_event_info()`, `normalize_meal()` — handle flexible/variant Excel column names and meal type abbreviations (e.g. BF → Breakfast)

**Excel I/O:** `read_event_info()`, `read_menu_data()`, `read_meal_counts()`, `ensure_meal_counts_sheet()`, `create_template_excel()`, `reset_excel()`

**Menu building:** `build_menu_tree()` → hierarchical date/meal/category structure; `build_date_pages()` → splits meals into left/right columns for PDF layout

**PDF generation:** `generate_menu_pdf(lang)` for a single language; `generate_menu_pdfs()` for both EN + HI

### Excel Sheet Structure

**event_info:** `key | value` rows — fields like `event_name`, `client_name`, `venue`, `start_date`, `end_date`, `caterer_name`, `notes_text`, and Hindi variants (`*_hi`)

**menu:** `date | meal | category | item` — dates/meals auto-populate via Excel formulas; users fill categories and items

**meal_counts:** `date | meal | count` — auto-generated by `ensure_meal_counts_sheet()`; users enter pax counts

### Bilingual Support
- All output generated in English and Hindi
- Hindi text uses `assets/NotoSerifDevanagari-Regular.ttf` via WeasyPrint
- Translations defined as constants in `menu_generator.py` (`*_HI` dicts)
- `format_date_localized()` handles Hindi month names

### Platform Notes
- **macOS:** GTK3 not required (WeasyPrint uses CoreGraphics)
- **Windows:** GTK3 Runtime must be installed; `app.py` dynamically locates the GTK3 runtime path (checks bundled path first, then system)
- Build scripts use PyInstaller; GTK3 is intentionally NOT bundled on Windows to reduce executable size
